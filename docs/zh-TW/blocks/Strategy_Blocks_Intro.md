# 核心理念：量化策略積木 (Strategy Blocks)

> **"Stop reinventing the wheel. Start assembling the car."**  
> 不要重複造輪子，開始組裝跑車。

---

## 1. Why? 為什麼我們需要「策略積木」？

在過去的開發經驗中，我們發現團隊成員在寫策略時，常常陷入 **「從零開始 (From Scratch)」** 的泥淖。

每次寫新策略都要重新思考：
* 資料怎麼抓？
* 回測怎麼設？
* 交易怎麼執行？

這不僅 **效率低落**，更容易產生重複的 **Bug**。為了實現規模化的量化研究，我們將過去驗證成功的 **9 支策略**，解構為可重複使用的 **「標準化積木」**。

---

## 2. Value Proposition：積木化的三大好處

### 🚀 極致效率 (Efficiency)
開發新策略不再需要 3 天，**只需要 3 小時**。只需替換核心邏輯模塊，骨架直接套用。

### 🛡️ 邏輯防呆 (Robustness)
經過壓力測試的標準架構，能避免常見的低級錯誤：
* **倖存者偏差 (Survivorship Bias)**
* **前視偏差 (Look-ahead Bias)**
* **資料洩漏 (Data Leakage)**

### 💬 協作語言 (Common Language)
當所有人都用同一套架構溝通，溝通成本將大幅降低。
> *Scenario：「我用 **Pipeline因子架構**，CustomDataset 那邊怪怪的...」*（秒懂）

## 3. SOP：如何使用本手冊？

本手冊將量化開發歸納為 **三大架構**。在開始寫 Code 之前，請先靈魂拷問自己一個問題：

> **「我的獲利邏輯是基於財報選股？還是技術指標？還是全市場掃描？」**

請根據你的邏輯，選擇對應的架構：

| 你的策略邏輯 | 推薦架構 | 適用場景 | 代表策略 |
| :--- | :---: | :--- | :--- |
| **選股 / 財報** | **`財報選股架構`** | 基本面分析、季報選股 | 多因子選股、Dreman逆向投資 |
| **技術 / 個股** | **`技術指標架構`** | 技術指標、價量分析 | MACD、布林通道 |
| **籌碼 / 全市場** | **`Pipeline因子架構`** | 籌碼面、Market Scan | Momentum、跟隨大戶 |

**👉 Next Step:**  
選定架構後，請直接複製對應頁面的 **Code Template**，填入你的獨家因子，即可完成策略開發。

## 4. 策略案例索引 (Strategy Index)

本手冊基於以下 9 個實戰策略解構而成：

### 📊 財報選股架構 (3個)
1. **多因子選股** - 本益比、流動比率、現金股利率組合
2. **小型成長股** - 市值<30% + 淨利成長≥15% + PEG<1  
3. **Dreman逆向投資** - 兩階段篩選 + 計分制

### 📈 技術指標架構 (3個)
4. **MACD** - 快慢線交叉 + 柱狀圖轉折
5. **乖離率 (BIAS)** - 7日EMA偏離 + 高低點突破
6. **布林通道** - 20日通道 + 反轉交易

### 🔬 Pipeline因子架構 (3個)
7. **Expanded Momentum** - 迴歸動能分數 + 反向波動度加權
8. **跟隨大戶** - 三大法人籌碼 + 持股率均線
9. **CounterTrend** - 趨勢過濾 + 逆勢買跌
